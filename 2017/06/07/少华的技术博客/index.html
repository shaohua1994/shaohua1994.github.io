使用gulp创建前端自动化工作流环境

分类: 插件&工具 node.js 学习笔记

什么是gulp ?

  gulp是前端开发过程中对代码进行构建的工具，是自动化项目的构建利器；她不仅能对网站资源进行优化，而且在开发过程中很多重复的任务能够使用正确的工具自动完成；使用她，我们不仅可以很愉快的编写代码，而且大大提高我们的工作效率。
  gulp是基于Nodejs的自动任务运行器， 她能自动化地完成 javascript/coffee/sass/less/html/image/css 等文件的的测试、检查、合并、压缩、格式化、浏览器自动刷新、部署文件生成，并监听文件在改动后重复指定的这些步骤。在实现上，她借鉴了Unix操作系统的管道（pipe）思想，前一级的输出，直接变成后一级的输入，使得在操作上非常简单。

  在学习前，先谈谈大致使用gulp的步骤，给读者以初步的认识。首先当然是安装nodejs，通过nodejs的npm全局安装和项目安装gulp，其次在项目里安装所需要的gulp插件，然后新建gulp的配置文件gulpfile.js并写好配置信息（定义gulp任务），最后通过命令提示符运行gulp任务即可。

安装nodejs -> 全局安装gulp -> 项目安装gulp以及gulp插件 -> 配置gulpfile.js -> 运行任务

安装node可以去github官网下载然后直接安装就可以,打开命令行输入node -v 检查是否安装成功。因为新版的nodejs内置有npm所以不需要再安装npm.npm 是nodejs 的包管理工具,通过它可以下载安装nodejs的模块包。完成之后我们可以来下载gulp工具了。

首先我们来搭建gulp工程:

gulp/
  ├── src/
  │   ├── less/
  │   │   └─ main.scss
  │   ├── scripts/
  │   │   └─ javascript.js
  │   └── index.html
  ├── dist/
  └── gulpfile.js /

1.先全局安装gulp

  • 说明：全局安装gulp目的是为了通过她执行gulp任务；

    npm install gulp -g

  • 查看是否正确安装：命令提示符执行gulp -v，出现版本号即为正确安装。

    gulp -v

  • 然后回到文件根目录(gulp)命令行输入

    npm init        //创建一个package.json文件

2.进行本地安装gulp 命令行行输入

npm install gulp --save-dev

gulp常用组件

gulp-htmlmin --save

gulp-less --save   

gulp-cssnano       

gulp-uglify

gulp常用API

gulp.task('任务名称',['其他任务'],回掉函数)

gulp.src('文件路径')

gulp.dest('目标路径')   

gul.watch   //监听   

.pipe   //起到一个管道作用

3.然后可以进行安装gulp插件

npm install gulp-htmlmin --save     //gulp-htmlmin插件对html文件进行压缩

npm install gulp-less --save    //对less文件的编译成为css文件

npm install gulp-cssnano        //压缩css文件

npm install gulp-uglify         //对js文件压缩

npm install browser-sync           //让浏览器实时、快速相应文件的更改，并自动刷新页面

接下来可以在gulpgile.js文件里进行编写

首先要引入需要的几个插件

对html进行压缩操作

完成后可以命令行输入 gulp-htmlmin
此时可以看html文件已经压缩

对less文件进行编译

命令窗口输入 gulp less

对js进行压缩

命令窗口输入 gulp js

分别开启监听和服务 (命令窗口输入hexo watch )(hexo s)

或者直接定义一个默认任务,调用watch和serve,监听所有的文件

命令行只输入gulp //会执行所有任务

最后一步,打开页面同时在编辑器里修改代码页面会同时改变,证明已经成功了!
